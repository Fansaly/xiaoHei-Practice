The purpose of this guide is to provide a step-by-step guide to installing High Sierra, Sierra, and El Capitan on the Lenovo Z50-70 and the Z40-70 too.<br><br>
My machine is a Z50-70 with Broadcom BCM943602BAED and SSD.<br><br>
The laptop BIOS implements a WiFi whitelist, which must be defeated in order to install an OS X compatible WiFi card. Another possibility is to use AR9565 that comes with some of the Z50-70/Z40-70 machines which is already whitelisted in the BIOS. Installation scripts have been updated to install the appropriate kext for the installed Wi-Fi card.<br><span style="color: rgb(0, 0, 0)"><br>
If you are installing Sierra or El Capitan, it is expected that you will read the guide here, but ask questions at the El Capitan thread. This thread should be used only for High Sierra questions.<br><br><b>What you need</b><br><br>
- Lenovo Z50-70 or Z40-70<br>
- macOS High Sierra, Sierra, or El Capitan downloaded from the Mac App Store<br>
- 8GB USB stick<br>
- (preferred) Broadcom BCM943602BAED or BCM94352Z for native WiFi/BT</span><br><span style="color: #000000">- Atheros AR9565 (Using Lilu.kext + ATH9KFixup.kext) for native WiFi</span><br><span style="color: rgb(0, 0, 0)"><br><br><b>BIOS settings</b><br></span><br>
To start, set BIOS to Windows defaults.<br><br>
Then insure:<br>
- UEFI boot is enabled<br>
- secure boot is disabled<br>
- enable Legacy Boot (but UEFI first) and you may experience less boot time glitches<br><br>
Note: If your laptop has a discrete graphics card (Nvidia), keep it enabled in BIOS. It will be disabled automatically by SSDT-NVDA.aml when running OS X.<br><span style="color: rgb(0, 0, 0)"><br><br><b>Preparing USB and initial Installation</b><br><br>
This guide for creating USB and installing using Clover UEFI works well for this laptop: <a href="http://www.tonymacx86.com/el-capitan-laptop-support/148093-guide-booting-os-x-installer-laptops-clover.html" class="internalLink">http://www.tonymacx86.com/el-capita...de-booting-os-x-installer-laptops-clover.html</a><br><br>
Special notes:<br><br>
- Use the 'createinstallmedia' approach. It works well, and there is little chance for pilot error. This method also gives you an OS X recovery partition. Follow RehabMan's guide here &gt; <a href="http://www.tonymacx86.com/el-capitan-laptop-support/148093-guide-booting-os-x-installer-laptops-clover.html" class="internalLink">[Guide] Booting the OS X Installers on LAPTOPS with Clover</a>.</span><br><br><span style="color: rgb(0, 0, 0)">- Definitely copy RealtekRTL8111.kext to Clover/kexts/Other as having network support during post-install is helpful. The rest of this guide depends on it. An alternate is to copy FakePCIID.kext and FakePCIID_Broadcom_WiFi.kext to Clover/kexts/Other or Lilu.kext and ATH9KFixup.kext+ATH9KInjector.kext if you have AR9565. This will enable WiFi, provided you have a compatible WiFi card already installed.</span><br><br>
Note: FakePCIID.kext and FakePCIID_Broadcom_WiFi.kext are available here: <a href="https://github.com/RehabMan/OS-X-Fake-PCI-ID" target="_blank" class="externalLink ProxyLink" data-proxy-href="proxy.php?link=https%3A%2F%2Fgithub.com%2FRehabMan%2FOS-X-Fake-PCI-ID&amp;hash=c3419c0e84ef33abff1f428a88efa756" rel="nofollow">https://github.com/RehabMan/OS-X-Fake-PCI-ID</a>, Lilu.kext is available here: <a href="https://github.com/RehabMan/Lilu" target="_blank" class="externalLink ProxyLink" data-proxy-href="proxy.php?link=https%3A%2F%2Fgithub.com%2FRehabMan%2FLilu&amp;hash=1076f1f0be346c938313ef25041cba45" rel="nofollow">https://github.com/RehabMan/Lilu</a>, and ATH9KFixup.kext and ATH9KInjector.kext are available here: <a href="https://github.com/RehabMan/ATH9KFixup" target="_blank" class="externalLink ProxyLink" data-proxy-href="proxy.php?link=https%3A%2F%2Fgithub.com%2FRehabMan%2FATH9KFixup&amp;hash=24ba423c0aadee49e9632b55e34e2c4c" rel="nofollow">https://github.com/RehabMan/ATH9KFixup</a>.<br><br><span style="color: rgb(0, 0, 0)">- Some of these laptops come with ELAN trackpad so RehabMan's VoodoPS2Controller.kext might not work properly. An alternate is to download ELAN kext from the Z50 repo.</span><br><br>
- GenericUSBXHCI.kext is not necessary with this laptop since AppleUSBXHCI.kext appears to work well natively. Also, GenericUSBXHCI.kext does not work on 10.11+ so it should not be used.<br><br><br>
- Use the config_install.plist from the z50 repo: <a href="https://github.com/the-braveknight/Lenovo-Z50-DSDT-Patch/raw/master/config_install.plist" target="_blank" class="externalLink ProxyLink" data-proxy-href="proxy.php?link=https%3A%2F%2Fgithub.com%2Fthe-braveknight%2FLenovo-Z50-DSDT-Patch%2Fraw%2Fmaster%2Fconfig_install.plist&amp;hash=e21b044cd6e3ff11eb685987136d0d70" rel="nofollow">https://github.com/the-braveknight/Lenovo-Z50-DSDT-Patch/raw/master/config_install.plist</a><br><span style="color: rgb(0, 0, 0)"><br><br><b>Post Installation</b><br></span><br>
Install Clover UEFI as described in the guide linked by the previous section (post #2). After installing Clover, and configuring it correctly (config.plist, kexts, etc) you should be able to boot from the HDD/SSD.<br><br>
But there are still many issues and devices that won't work correctly. For that, we need to patch DSDT, provide a proper config.plist, and install the kexts that are required.<br><br>
Since you have RealtekRTL8111.kext already injected by Clover, you should have internet access simply by using an Ethernet cable to your router. Plug it in and make sure you have internet access before continuing. Or if you're using FakePCIID_Broadcom_WiFi.kext, you can connect to your WiFi router before continuing.<br><span style="color: rgb(0, 0, 0)"><br>
Installation of the tools and patching is easy provided the scripts and tools at the repository: <a href="http://github.com/Zaid-Rahawi/Lenovo-Z50" target="_blank" class="externalLink ProxyLink" data-proxy-href="proxy.php?link=http%3A%2F%2Fgithub.com%2FZaid-Rahawi%2FLenovo-Z50&amp;hash=5309cfc8170279e5fcea4c64597c875b" rel="nofollow">http://github.com/the-braveknight/Lenovo-Z50</a><a href="http://github.com/the-braveknight/Lenovo-Z50-DSDT-Patch" target="_blank" class="externalLink ProxyLink" data-proxy-href="proxy.php?link=http%3A%2F%2Fgithub.com%2Fthe-braveknight%2FLenovo-Z50-DSDT-Patch&amp;hash=09dc5feca3dbaf9e833219fc846fefff" rel="nofollow">-DSDT-Patch</a><br><br>
To start, the developer tools must be installed. Run Terminal, and type:<br><div class="bbCodeBlock bbCodeCode"><div class="type">Code (Text):</div><div class="code"><br>
git<br>
&nbsp;</div></div>You will be prompted to install the developer tools. Since you have internet working, you can choose to have it download and install them automatically. Do that before continuing.<br>
After the developer tools are installed, we need to make a copy of the project on github.<br><br>
In Terminal:<br><div class="bbCodeBlock bbCodeCode"><div class="type">Code (Text):</div><div class="code"><br>
mkdir ~/Projects<br>
cd ~/Projects<br>
git clone https://github.com/the-braveknight/Lenovo-Z50-DSDT-Patch.git z50.git<br>
&nbsp;</div></div>Now it is time to install some more tools and all the kexts that are required...<br>
To download the kexts &amp; tools, in Terminal:<br><div class="bbCodeBlock bbCodeCode"><div class="type">Code (Text):</div><div class="code"><br>
cd ~/Projects/z50.git<br>
./download.sh<br>
&nbsp;</div></div>To install:<br><div class="bbCodeBlock bbCodeCode"><div class="type">Code (Text):</div><div class="code"><br>
./install_downloads.sh<br>
&nbsp;</div></div></span><br><br>
The download.sh script will automatically gather the latest version of all tools (patchmatic, iasl, MaciASL) and all the kexts (IntelBacklight.kext, ACPIBatteryManager.kext, etc) from bitbucket. The install_downloads.sh will automatically install them to the proper locations.<br><br>
If you wish, you can reboot to verify a few more items are working (many only partially).<br><br>
To finish the setup, we need a correctly patched DSDT/SSDT.<br><br>
With the current project, no patched DSDT/SSDTs are used. Instead I use Clover hotpatches and a group of small SSDTs.<br><br><span style="color: rgb(0, 0, 0)">In Terminal:<br><div class="bbCodeBlock bbCodeCode"><div class="type">Code (Text):</div><div class="code"><br>
cd ~/Projects/z50.git<br>
make<br>
make install<br>
&nbsp;</div></div>The 'make' causes the patched files to be compiled (with iasl), the results placed in ./build.<br></span><br>
Finally, 'make install', mounts the EFI partition, and copies the built files where they can be loaded by Clover (to EFI/Clover/ACPI/patched).<br><br><b>Power Management</b><br><br>
Everything required for CPU/IGPU power management is already installed with the steps above.<br>
There is no longer any need to use the ssdtPRgen.sh script.<br><br>
Also, be aware that hibernation (suspend to disk or S4 sleep) is not supported on hackintosh.<br><br>
You should disable it:<br><span style="color: rgb(0, 0, 0)"><div class="bbCodeBlock bbCodeCode"><div class="type">Code (Text):</div><div class="code"><br>
sudo pmset -a hibernatemode 0<br>
sudo rm /var/vm/sleepimage<br>
sudo mkdir /var/vm/sleepimage<br>
&nbsp;</div></div></span><br>
Always check your hibernatemode after updates and disable it. System updates tend to re-enable it, although the trick above (making sleepimage a directory) tends to help.<br><span style="color: rgb(0, 0, 0)"><br><b>Final config.plist</b></span><br>
Up to now, you've been using the same config.plist we were using for installation. After all the APCI files are in place (previous two steps), you're ready to use the final config.plist from the z50 repo.<br><br>
First, mount the EFI partition:<br><span style="color: rgb(0, 0, 0)"><div class="bbCodeBlock bbCodeCode"><div class="type">Code (Text):</div><div class="code"><br>
cd ~/Projects/z50.git<br>
./mount_efi.sh /<br>
&nbsp;</div></div></span><br>
Then copy the file:<br><span style="color: rgb(0, 0, 0)"><div class="bbCodeBlock bbCodeCode"><div class="type">Code (Text):</div><div class="code"><br>
cd ~/Projects/z50.git<br>
cp config.plist /Volumes/EFI/EFI/Clover/config.plist<br>
&nbsp;</div></div></span><br><br>
After copying the config.plist from the repo to EFI/Clover/config.plist, you should customize the SMBIOS so you have a unique serial. You can use Clover Configurator to do this (use google to find/download it). DO NOT use Clover Configurator to edit your actual config.plist. Instead edit a "dummy" config.plist to create the SMBIOS data and then use copy/paste with a plist editor (I use Xcode) to copy the SMBIOS section into my active config.plist. Clover Configurator is too buggy and cannot be trusted with edits to your real config.plist. This guide uses MacBookAir6,2. Do not use any other model identifier.<br><br>
Repeat: <b>DO NOT edit your config.plist with Clover Configurator</b>. Clover Configurator will erase important settings from the config.plist, and as a result, it will not work.<br><br><br><b>Do not stop reading</b><br><br>
Although most of the post-install tasks are done, continue to read this guide. It it has important information you should know about.<br><br>
Be sure to visit "Known Problems", especially the instructions in the "Audio:" section. Typically, even after completing all the above tasks, audio will not work on the first reboot. You must follow the specific instructions in that section in order to get the audio kexts in cache.<br><br>
In the case of a problem, don't bother asking about with without all files requested in "Problem Reporting".<br><br><br><b><span style="color: rgb(0, 0, 0)">BIOS mods for compatible WiFi installation</span></b><br><span style="color: rgb(0, 0, 0)"><br>
The laptop BIOS implements a WiFi whitelist, which must be defeated in order to install an OS X compatible WiFi card.<br><br>
See this guide for more info: <a href="http://www.tonymacx86.com/el-capitan-laptop-support/187340-guide-lenovo-g50-70-z50-70-bios-whitelist-removal.html" class="internalLink">http://www.tonymacx86.com/el-capita...ovo-g50-70-z50-70-bios-whitelist-removal.html</a></span><br><br><br><span style="color: rgb(0, 0, 0)"><b>WiFi using BCM94352Z</b><br><br>
The ./download.sh and ./install_downloads.sh script have both been updated to install the necessary kexts for BCM94352Z.<br><br>
The config.plist contains the patches required for WiFi 5Ghz and Bluetooth handoff features. The patch in config.plist for 5GHz is the US/FCC patch by the-darkvoid. If you're not in the US, you might consider a different patch. Read here: <a href="http://www.tonymacx86.com/network/104850-guide-airport-pcie-half-mini-v2.html" class="internalLink">http://www.tonymacx86.com/network/104850-guide-airport-pcie-half-mini-v2.html</a><br><br></span><br><b><span style="color: rgb(0, 0, 0)">WiFi using BCM943602BAED (DW1830)</span></b><br><br><span style="color: rgb(0, 0, 0)">This card is a bit more native as it uses the same chip that's found in original Macs (Bcm4360), it is recognized OOB, FakePCIID is used to get the AirPort branding. It doesn't require the 5GHz patch or the fcvo patch that's is required for Bcm4352 cards in macOS Sierra (10.12).</span><br><br><br><b>WiFi using Atheros AR9565</b><br><br>
Initial support was added for the Atheros AR9565 Wi-Fi card that comes with some of the Z50-70/Z40-70 laptops using ATH9KFixup.kex for those who do not want to extract/patch their BIOS.<br><br>
Installation scripts have been modified to install the necessary kexts for Broadcom (BCM94352Z or BCM943602BAED) &amp; Atheros AR9565 cards.<br><br><br><span style="color: rgb(0, 0, 0)"><b>Updates to the patch repositories</b><br><br>
From time to time, updates may become available to the project repository. In the event of such updates, you may want to update your copies, and re-patch DSDT/SSDT with the updates.<br><br>
Since you're using git, it is easy...<br><br>
In Terminal:<br><div class="bbCodeBlock bbCodeCode"><div class="type">Code (Text):</div><div class="code"><br>
cd ~/Projects/z50.git<br>
git pull<br>
./download.sh<br>
./install_downloads.sh<br>
./update_config.sh<br>
make<br>
make install<br>

Find My Mac/Locking: Find My Mac does not work properly. Don't lock your mac because it's difficult (or impossible) to unlock again.<br><br>
Slow WiFi after sleep/wake cycle: Disable "Wake for network access" in SysPrefs-&gt;Energy Saver.<br><br>
Audio: After installing new/updated kexts or after the initial installation, you may lose audio.<br><br>
To fix:<br>
- reboot without caches (hit space at Clover, select without caches)<br>
- rebuild cache<br><div class="bbCodeBlock bbCodeCode"><div class="type">Code (Text):</div><div class="code"><br>
sudo touch /System/Library/Extensions &amp;&amp; sudo kextcache -u /<br>
&nbsp;</div></div>- reboot normally (if needed, two times)<br><br>
Audio: External mic via combo headphone/mic jack is not working.<br><br>
CPU Frequency: Clover identifies the CPU speed (as shown in "About this Mac") incorrectly. On my system (2.0Ghz Core i7-4510u), it shows 2.59Ghz. This appears to be cosmetic. You can fix it by overriding Clover logic with config.plist/CPU/FrequencyMHz. You will see how my setting is commented in the config.plist. Set it as appropriate for your own CPU.<br><br><br><span style="color: rgb(0, 0, 0)"><b>Other post-install tasks</b><br><br>
Trackpad: Be sure to visit the options in SysPrefs-&gt;Trackpad and change them to your liking.<br><br>
Bluetooth: If you get the Bluetooth Setup Assistant popup, go to SysPrefs-&gt;Bluetooth-&gt;Advanced, uncheck the boxes.<br><br><br><b>Updating to High Sierra</b><br><br>
As you probably already know, High Sierra has a new file system called APFS. Boot drives on SSDs will automatically be converted to APFS if you start the High Sierra installer in the default way (eg. running /Applications/Install macOS High Sierra.app).<br><br>
If you intend to use APFS, don't forget to add apfs.efi to EFI/CLOVER/drivers64UEFI. Without <b>apfs.efi in drivers64UEFI, Clover will not recognize APFS</b> boot volumes. You can find apfs.efi at /usr/standalone/i386/apfs.efi inside of "/Applications/Install macOS High Sierra.app/Contents/SharedSupport/BaseBinaries.dmg".<br><br>
But you can avoid converting to APFS if you want to stick with HFS+. To so so, don't use the Install macOS High Sierra.app to start the installer. Use 'startosinstall' instead:<br></span><br><span style="color: #000000"><div class="bbCodeBlock bbCodeCode"><div class="type">Code (Text):</div><div class="code"><br>
/Applications/"Install macOS High Sierra.app"/Contents/Resources/startosinstall --converttoapfs NO<br>
